Architecture

In a nutshell:

  • Most Account Information Service Providers have REST (not GraphQL) APIs. We convert their output to flow through our systems as schema-based GraphQL JSON.
  • Flow modules (a flavor of microservices) collaborate in the backend to perform integration, normalization, aggregation, and analysis tasks. These modules are managed by Spring Cloud Data Flow, itself running on Kubernetes in the Google Cloud.
  • We use the Netflix Domain Graph Service (DGS) to expose backend data to the frontend user interface, where the Apollo client is used in place of e.g. Redux to serve data into React components. Capacitor is used to create a mobile client.
  • OAuth2 (specifically Open ID) are the standard method of authenticating on both front and back-end.

Work-in-progress architecture as-built Sep 2021.

Data
